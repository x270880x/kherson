# Отчёт о качестве геометрии данных

Автоматический аудит всех полигонов участков. Метод: площадь, вычисленная из геометрии, сравнивается с заявленной площадью `a` из `councils/*.json` (при обрезке контура `a` остаётся верной). Локальная равнопромежуточная проекция вокруг широты совета, формула шнурков. Медианное отношение по норме ≈ **0.99** (геометрия совпадает с декларацией).

- **55.5%** обрезанных участков имеют ось-параллельную прямую кромку (горизонтальную/вертикальную) — признак клипа по прямой линии.
- Одна и та же широта **46.702261** срезает участки сразу в нескольких разных советах (напр. Надіївка — 28 уч.) → это **глобальная линия сетки/бокса в исходных данных**.

## Прочие дефекты

- **«Сироты» (972):** полигон есть на карте, но нет ни одной атрибутивной записи (проверено — ни в одном совете). На карте отображаются, в попапе — «нет данных в базе». Это пробел полноты данных (скрейпа), не ошибка раскладки. 727 из них — в блоке `:001` (застройка).
- **Без геометрии (97):** участок есть в атрибутах (учитывается в статистике и площади совета), но полигон не нарисован → невидим на карте. Концентрация: Степне (38), Верби (30) — север области.
- **Без площади (1166):** `a`=0/пусто, форма собственности «Не визначено» — неполные записи реестра.

## Что НЕ является дефектом

- Дубликатов кадастровых номеров нет. Вырожденных полигонов нет. Координат вне области нет.

## Рекомендации

1. Обрезанные контуры локально не чинятся — нужен корректный экспорт геометрии из источника. Площадь брать из `a`.
2. Опционально: флаг «⚠ контур обрезан» в попапе участка (по `geom/a` < 0.5), чтобы пользователь видел неполноту отрисовки.
3. «Сироты» и пропуски геометрии — устранятся только при полном переимпорте из достоверного источника.
