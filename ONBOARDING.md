## Как запустить (preview)
Сервер настроен в `~/.claude/launch.json` под именем **`kherson`** (python `serve.py`, порт **8773**, корень `~/Documents/kherson`).
1. `preview_start` с `name: "kherson"` (НЕ через Bash).
2. Проверять: `preview_eval` (состояние), `preview_screenshot` (вид), `preview_console_logs` (ошибки).
3. **Важно:** preview-окно бывает узким (мобильная ширина). Для десктоп-проверки `preview_resize` на 1440×900; для мобильной — preset `mobile` (375).
4. `preview_click` ненадёжен для Leaflet-контролов (шестерёнка) — поведение проверять через эмуляцию событий (`dispatchEvent`/`map.fire`), это эквивалент реального клика.
5. **Бета** открывается по `http://localhost:8773/map-beta.html`. Гоча: иногда свежий preview-инстанс **не рисует MapLibre** (пустой экран, `map.getStyle()` пуст) — это баг инстанса, не кода (заведомо рабочая версия тоже пустая). Лечится `preview_stop` + `preview_start` (новый сервер). Однопоточный `serve.py` медленно отдаёт тяжёлые тайлы беты — для оценки скорости смотреть живой сайт (CDN), не локалку.

## Структура
- **`index.html`** (~499 КБ после оптимизации) — весь сайт в одном файле:
  - `<style>` в `<head>` (включая `@media (max-width: 768px)` — мобильная версия);
  - `<script id="data">` — встроенный JSON: сводки **только 337 херсонских советов** (code, name, lat, lon, region, total, area, by_ownership, top_renters/owners/categories) + `stats`, `filters`, `renter_councils`. `STATS = DATA.stats` (используется поиском по арендатору). Рантайм-фильтр `COUNCILS = DATA.councils.filter(region==='Херсон')` — теперь по сути no-op.
  - основной JS в конце `<body>`.
- **`parcels/*.geojson`** (348 файлов, ~583 340 участков) — геометрия участков (в properties только `cadnum`), ленивая подгрузка по коду совета. Не-херсонские удалены. **2026-06-20:** (1) геометрия сверена со снапшотом kadastrova-karta.com.ua — починено 9 501 обрезанных, добавлено 25 756 пропущенных (`src:"snapshot2026"`); (2) добавлен весь Нововоронцовский район (КОАТУУ 65241): смт + 12 сёл, ~31 686 участков **с атрибутами из vkursi** (`src:"vkursi2026"`). См. CHANGELOG и [[project_kherson_refetch]]. Тулинг — `.wip/refetch/` (gitignored).
- **`councils/*.json`** (336 файлов) — атрибуты участков совета (`c`,`a`,`o`,`r`,`ot`,`ct`,`mz`…), ленивая подгрузка. Не-херсонские удалены. Новые участки — стабы с пустыми атрибутами.
- **Внешние ресурсы:** Leaflet+markercluster+**Leaflet.VectorGrid** (unpkg, в `<body>`), шрифты IBM Plex (Google Fonts, async-`<link>` без render-blocking), **SheetJS — ленивая загрузка** `loadXlsx()` только при первом экспорте в Excel (не на старте). В `<head>` — preconnect/dns-prefetch к этим доменам и тайлам.
- **`tiles/{z}/{x}/{y}.pbf`** (~124 МБ, 11.5к файлов, zoom 8–14) — **векторные тайлы всех 530к участков**. Единственное свойство — `o` (форма собств. p/k/g/n) для раскраски; кадастр/площадь/данные участка берутся при клике из оригинальных `parcels/`+`councils/` (point-in-polygon), не из тайлов. Лимит тайла 2 МБ → с **z9 и выше участки НЕ прореживаются** (карта залита на обзоре; режет только z8 — вся область в паре тайлов). Генерятся `build_tiles.sh` (tippecanoe, без сжатия — GitHub Pages отдаёт raw .pbf). Используются режимом **«Полная карта»** (шестерёнка).
- **`build_tiles.sh`** — пересборка `tiles/` из `parcels/`+`councils/` (требует tippecanoe). Запускать при изменении данных участков.
- **`map-beta.html`** — **бета-карта на MapLibre GL** (отдельная страница, ссылка «🧪 Бета-карта» из левой панели index.html; не рантайм-тумблер движков, основной Leaflet-код не трогает). Почти паритет с основным приложением:
  - GPU-рендер всех участков из `tiles/` (плавно на любом зуме); цвет по `o`; `fill-antialias:false`, `fadeDuration:0`; свои кнопки +/− (мгновенный зум).
  - **3 подложки + переключатель в HUD:** «Светлая» (CARTO Voyager `rastertiles/voyager_nolabels`, **по умолчанию** — видно дороги/контекст), «Тёмная» (CARTO `dark_nolabels`), «Спутник» (Esri World Imagery). Источники `light`/`base`/`sat`, `setBase()` переключает visibility.
  - **Русские подписи** (CARTO без подписей + OpenFreeMap `omt`, `name:ru` через coalesce, глифы OpenFreeMap). Слои: `lbl-water` (гидронимы), `lbl-place` (НП OMT — `county/city/town/village/hamlet`), `lbl-state` (**соседние области**, class `state`, вдвое мельче и тусклее — `text-opacity 0.5`; Херсонская исключена по имени, у неё кастомный титул), **`lbl-councils`** (подписи наших **сельсоветов — пгт/сёла** из `councils-index.json`; OMT на обзорном зуме сёла не отдаёт. Источник `councils` geojson, видим с z8, `symbol-sort-key` по числу участков, collision; добавлен поверх `lbl-place` → OMT-города в приоритете, наши заполняют пробелы. `addCouncilLabels()` на `styledata`). `setBaseLabels()` адаптирует цвет текста `lbl-place/lbl-councils/lbl-state/lbl-water` под фон.
  - **Наведение на НП → табличка форм собственности** (`hoverPop`, `ownTableHTML`): hover по `lbl-place` или `lbl-councils` → ближайший совет (`nearestCouncil`, в пределах ~22км) → попап со счётчиками участков Частная/Коммунальная/Государственная/Не определено + всего (цвета `OWN`). Попап `pointer-events:none` (не блокирует клик). **Клик по НП** (`lbl-place` и `lbl-councils`) → приближение (район `fitBounds` 1600мс / село `fitBounds` соседних 700мс). Guard в обработчике клика по `parcels-fill` пропускает клики по обеим подписям.
  - Вкл/выкл форм собственности на всей карте (вкл. частные — фильтр MapLibre `setFilter` по `o`), клик→полные данные участка (point-in-polygon по `parcels/` + ленивая `councils/`), поиск (село/кадастр/арендатор), подборка (Shift-клик/рамка, подсветка geojson-слоем, CSV), **тонкие фильтры по гибриду** (категория/год аренды/межевание/арендатор: при zoom≥13 тайлы прячутся, оверлей `fine` показывает только подходящих из подгруженных данных совета), шапка-статистика. Атрибуция убрана (`attributionControl:false`).
  - Данные для беты: **`councils-index.json`** (337 советов: `c/n/lat/lon` + **`o:{p,k,g,n}`** счётчики участков по формам + `total` — для клика/поиска/подписей/таблички наведения; поле `o`/`total` дополняется скриптом из `<script id="data">` index.html, остальные поля не трогать), **`filters-beta.json`** (категории, годы аренды, арендаторы + `renter_councils` для перелёта).   - **Титул региона** (поверх карты): герб + «Херсонская область» — фиксированный CSS-оверлей (`#region-title`/`.rt-inner`, НЕ гео-маркер), по центру экрана крупно (текст 27px, герб 40px). ## Ключевые места в коде (index.html)
- **Карта:** Leaflet 1.9.4 + markercluster, `preferCanvas: true`, `zoomSnap: 0`, `zoomDelta: 0.35` (мелкий шаг +/−), `maxZoom: 19`. Базовые слои: `baseDark` (OSM + CSS-инверсия `.map-tiles-dark`), `baseLight`, `baseSat` (Esri). По умолчанию — тёмный.
- **Слои:** `layersCtrl` свёрнут в иконку-шестерёнку (SVG); открытие по клику, закрытие по mousedown вне (hover отключён). 3 базовых слоя (радио) + overlay-чекбокс **«Полная карта (все участки)»** (фиктивный `fullMapLayer`, ловится `overlayadd`/`overlayremove`).
- **Режим «Полная карта»** (`fullMapMode`): показывает ВСЕ участки векторными тайлами (`L.vectorGrid.protobuf`, `tiles/`, renderer `L.canvas.tile`, `maxNativeZoom:14` + overzoom, цвет по `o`). Плавно на любом зуме. Маркеры-кружки скрыты; над сёлами в кадре — подписи (`fmAddLabel`/`fmLabelsUpdate`, до 60 советов, от зума 11). **Клик по участку — свой point-in-polygon** (`map.on('click')` → ближайшие советы → `rawParcels` + `geomContains`), т.к. canvas-тайлы кликать по объектам нельзя; попап лениво тянет полные данные из `councils/*.json`.
- **Маркеры/кластеры:** `buildMarkers()` (вызывается при каждой смене фильтра) → `clusterGroup`. В кластере — сумма участков (`m.pc`). Тултип с названием села привязан `direction: ttMobile ? 'bottom' : 'top'` (на мобиле — снизу, иначе перекрывался кружком и показывал только район). Обычный клик → `focusCouncil`; Shift+клик → `addCouncilView`.
- **Полигоны:** `loadCouncilPolygons(code, additive)` (lazy, кэш `polygonLayers`). Обычный клик → `showOnlyPolygons` (убирает ВСЕ прочие слои, показывает одно село). **ПК Shift+клик** по кружку → `addCouncilView` → `addPolygonsAdditive` (добавляет, НЕ закрывая ранее открытые; вписывает все показанные массивы в окно). `parcelStyle(f, council)` — **прячет участки, не подходящие под фильтр** (`hasAnyFilter()` + `parcelMatchesFilters`). При смене фильтра — `restyleFilteredPolygons()`.
- **Фокус на село:** `focusCouncil(code)` — `showDetail` + загрузка контуров + **плавный `flyTo`** (0.6с) на зум, вписывающий ВЕСЬ кластер участков в окно (центр смещён под правую панель). Bounds считаются через **`clusterBounds(layer)`** — отсекает географические выбросы (заборы Тьюки Q±3·IQR с «полом» 0.18°/0.22°): в данных у ~114 советов есть «улетевшие» участки, которые иначе раздувают bounds и кидают карту в центр области (напр. Східне `6520386700` — выброс на 156 км). Клик/поиск/список — все через `focusCouncil`. Анимация в preview не проигрывается (rAF на паузе в неактивной вкладке) — проверять мгновенным `setView`.
- **Район РФ:** `RAION_BY_PREFIX` (ключ = `code[2:5]`) → `raionOf(code)`. Сопоставлено с делением **РФ** (Указ 2023: 20 районов + 2 города Херсон/Новая Каховка) по громадам через обратное геокодирование. В данных — **17 районов + 2 города** (нет: Александровский, Высокопольский, Снигирёвский). Счётчик «районов» в шапке **захардкожен на 20** (официальное число РФ) по решению пользователя; тултипы при этом показывают фактический район по `raionOf`. Города: 101/201→Херсон, 107→Новая Каховка.
- **Названия:** `nameParts(name)` → `{ru, ua}` (рус из скобок + перевод префикса смт→пгт, м.→г.); `ruName(name)` (только рус); `detailTitle(c)` (рус крупно + «район (укр)» мелким). **22 совета вручную переименованы** (рус в скобках с перечислением сёл), напр. `6521280500` «Бериславський р-н» → «Рубановская громада (Демидовка, Запорожье…)», `6523884300` «Сірогозька сільрада» → «с. Сірогози (Серогозы)» (отдельное село, не Верхні/Нижні), `6523855000` «Нижні Сірогози (поля)» → «Нижні Сірогози — поля (Нижние Серогозы — поля общины)» (земли громады, не село), и т.д.; искались по «зависшим» агрегат-зонам без названия села — привязка через обратное геокодирование Nominatim (точки кластера) + сверка с укр. Wikipedia / decentralization.gov.ua.
- **Шапка-лента (hstats):** `.m`-спаны метрик c `<span class="sep">·</span>` + `&nbsp;` + `<wbr>`; id `hstat-parcels/area/councils/raions/mz`. Внутричисловые разрядные пробелы поджаты через `.hstats .v { word-spacing: -0.36em }` (mono-шрифт U+00A0 = целая ячейка), межметричные — `.sep { margin }`. На мобиле `#m-raions` и `#m-mz` (районы и «размежёвано») скрыты.
- **Фильтры и подсчёт:** `getFilteredStats(council)` — точный подсчёт, если `council.parcels` загружены (`parcelMatchesFilters`), иначе приближённый по агрегатам сводки. `buildMarkers()` прячет кружок, если под фильтром `count` не `> 0` (ловит 0 и NaN). Межевание (`filterMz`, РФ кадастр) нет в агрегатах сводки — отдельная карта **`MZ_COUNT`** `{код: число_размежёванных}` (105 советов, рядом с `getFilteredStats`); пересчитывается из `councils/*.json` (участки с непустым `mz`).
- **Формы собственности:** `OWN_COLOR`/`ownColor` (цвета: Частная #4caf7d, Коммунальная #e8c547, Государственная #4a9eff, Не определено #e05252); `OWNERSHIP_RU`/`ruOwn` (укр→рус только для отображения, данные не меняются); `OWN_SHORT`/`ruOwnShort`.
- **Подборка (basket):** `basket` (Set кадастров), `basketUpdate()`, `renderBasketPanel()` (правая панель со списком, площадями, итогами). Кадастр в списке красится `ownColor(r.ot)` по форме собственности (`recOf(cad)` достаёт участок). Рамочное выделение: Shift+drag (`setupBoxSelect`). Сброс: кнопка «Очистить подборку» (`btn-basket-clear`, левая панель) и «✕ Сброс» в правой панели.
- **Всплывающая плашка участка:** `buildParcelPopup` — рядом с УКР-кадастром и РФ-номером (mz) кнопки «скопировать» (`copyCad`/`copyBtn`/`fallbackCopy`). Размер плашки увеличен на 15%.
- **Итог по фильтру при нескольких открытых сёлах:** `openFilteredTotal()` + строка детал-статистики «Открыто сёл: N · всего по фильтру: X уч · Y га» (когда открыто >1 совета).
- **Правая панель** `aside.right` — выдвижная (классы `.open`), кнопки «›» свернуть / «‹» развернуть. На мобиле при выборе села НЕ открывается.
- **Мобайл (≤768px):** карта на весь экран; фильтры — выезжающая слева шторка `aside.left.open` по кнопке `#mobile-filters-btn` (☰, слева вверху карты); зум +/− скрыт.

### Бета-карта (`map-beta.html`) — известные проблемы / TODO (по фидбеку пользователя)
- **Полигоны участков не видны на стартовом кадре** — ✅ исправлено (`minZoom: 8` у карты). **Настоящая причина** (глубокий многоагентный разбор): источник `parcels` объявлен `minzoom: 8`, а стартовый `fitBounds(KHERSON_BOUNDS)` приземлялся НИЖЕ z8 на большинстве экранов (≈7.4–8.1 на ноутбуках, ~6.3 на мобиле; ≥8 только на 1920). MapLibre **не underzoom-ит** векторный источник ниже его `minzoom` (`Transform.coveringTiles`: `if coveringZoom < source.minzoom → return []`) → при covering-zoom 7 запрашивался НОЛЬ тайлов участков → `parcels-fill` рисовал пустоту. Плюс `map.once('idle', ()=>setMinZoom(getZoom()))` фиксировал камеру на под-z8. (В `index.html` те же тайлы рисуются, т.к. старт `setView(...,10)` всегда ≥8.) **Фикс:** `minZoom: 8` в конструкторе карты — `fitBounds` клампится вверх до z8, тайлы запрашиваются, участки рисуются с первого кадра. ⚠️ **Trade-off:** на экранах ýже ~1280px вся область при z8 чуть обрезается по краям; чтобы сохранить всю область на узких/мобильных — нужна **пересборка тайлов** (`tippecanoe -Z6`) + источник `minzoom:6` (одно снижение `minzoom` без новых тайлов = 404, т.к. MapLibre не underzoom-ит). **Не путать:** source `minzoom` (нижний зум ДАННЫХ тайлсета) ≠ map `minZoom` (нижний зум КАМЕРЫ). Тайлы/отдача/деплой/код-добавления-слоя — проверены и здоровы (декод: z8 плотно заполнен, 90 722 уч. в 7 тайлах).
